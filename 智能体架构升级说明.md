## 新增组件

### 1. DeadLoopDetector（防死循环检测器）
**位置**: `src/main/java/com/monuo/superaiagent/agent/detector/DeadLoopDetector.java`

**功能**:
- 检测语义重复
- 检测工具重复调用
- 检测连续失败
- 提供干预建议（PROMPT / WARNING / TERMINATE）

**使用方式**:
```java
DeadLoopDetector detector = new DeadLoopDetector();
detector.recordResponse("AI 回复内容");
DetectionResult result = detector.detect();
if (result.getLevel() == InterventionLevel.WARNING) {
    // 添加警告提示
}
```

---

### 2. ExecutionMonitor（执行监控器）
**位置**: `src/main/java/com/monuo/superaiagent/agent/monitor/ExecutionMonitor.java`

**功能**:
- 监控执行时间
- 检测超时
- 记录执行状态

**使用方式**:
```java
ExecutionMonitor monitor = new ExecutionMonitor();
monitor.start();
if (monitor.isTimeout()) {
    // 处理超时
}
```

---

### 3. ThinkingChain（思考链）
**位置**: `src/main/java/com/monuo/superaiagent/agent/thinking/ThinkingChain.java`

**功能**:
- 表示智能体的思考过程
- 包含思考内容、工具调用、回复内容

**数据结构**:
```java
ThinkingChain {
    QuestionType questionType;  // SIMPLE / COMPLEX
    String thinking;            // 思考内容
    boolean needTools;          // 是否需要工具
    List<ToolCall> toolCalls;   // 工具调用列表
    String directReply;         // 直接回复
}
```

---

### 4. ThinkingChainParser（思考链解析器）
**位置**: `src/main/java/com/monuo/superaiagent/agent/thinking/ThinkingChainParser.java`

**功能**:
- 解析 AI 输出的思考链格式
- 支持【思考】【工具调用】【回复】标记
- 支持 JSON 和文本两种格式

**支持格式**:
```
【思考】
分析用户意图...

【工具调用】
工具名称：webSearch，参数：{"query": "新闻"}

【回复】
已为您搜索到...
```

---

### 5. QuestionClassifier（问题分类器）
**位置**: `src/main/java/com/monuo/superaiagent/agent/thinking/QuestionClassifier.java`

**功能**:
- 快速判断问题是简单还是复杂
- 基于关键词匹配

**分类规则**:
- **简单问题**: 问候、感谢、简短提问（< 10字）
- **复杂问题**: 包含工具关键词（搜索、文件、邮件等）

---

### 6. ThinkingAgent（思考链智能体）
**位置**: `src/main/java/com/monuo/superaiagent/agent/ThinkingAgent.java`

**核心功能**:
- 实现混合思考模式
- 简单问题 → 直接回答
- 复杂问题 → 深度思考 + 工具调用

**工作流程**:
```
用户输入
    ↓
问题分类（QuestionClassifier）
    ↓
┌─────────────┬─────────────┐
│  简单问题    │  复杂问题    │
│  simpleThink │  deepThink  │
│  直接回答    │  思考链+工具 │
└─────────────┴─────────────┘
```

## 核心改进

### 1. 思考模式升级

**之前**:
- 只在检测到重复消息时才有【思考】【回复】格式
- 简单问候也尝试调用工具
- 缺少真正的推理过程

**现在**:
- 每次复杂问题都有完整的思考链
- 简单问题直接回答，不调用工具
- 明确的【思考】→【决策】→【回复】流程

### 2. 系统提示词优化

**之前**:
```
你是一个全能型AI助手...
核心能力：文件操作、网络搜索...
```
过于强调工具，导致总想着执行任务

**现在**:
```
你是 monuoManus，一个会深度思考的全能 AI 助手。

核心原则：
1. 先思考，再行动
2. 简单问题直接回答，复杂问题调用工具
3. 保持自然、友好的对话风格
```
强调思考优先，自然对话

### 3. 职责分离

| 组件 | 职责 |
|------|------|
| DeadLoopDetector | 防死循环检测 |
| ExecutionMonitor | 执行监控 |
| QuestionClassifier | 问题分类 |
| ThinkingChainParser | 思考链解析 |
| ThinkingAgent | 思考逻辑 |
| MonuoManus | 业务集成 |

## 使用示例

### 简单问题（无需工具）

**用户**: "你好"

**AI 处理流程**:
1. QuestionClassifier 判断为 SIMPLE
2. ThinkingAgent.simpleThink() 直接回答
3. 输出: "你好！我是 monuoManus，有什么可以帮你的吗？"

**特点**: 
- 快速响应
- 自然对话
- 不调用工具

---

### 复杂问题（需要工具）

**用户**: "帮我搜索今天的新闻"

**AI 处理流程**:
1. QuestionClassifier 判断为 COMPLEX
2. ThinkingAgent.deepThink() 生成思考链:
   ```
   【思考】
   用户想了解今日新闻，需要使用搜索工具获取最新信息
   
   【工具调用】
   webSearch
   
   【回复】
   正在为您搜索今日新闻...
   ```
3. 执行 webSearch 工具
4. 基于结果生成最终回复

**特点**:
- 有思考过程
- 明确工具调用
- 结果清晰

## 前端展示优化

### 思考过程展示

MonuoManus 已支持将【思考】和【回复】分开发送：

```java
// 发送思考内容（thinking 事件）
sseEmitter.send(SseEmitter.event().name("thinking").data(thinkingContent).build());

// 发送回复内容（message 事件）
sseEmitter.send(SseEmitter.event().name("message").data(replyContent).build());
```

**前端可以这样处理**:
```javascript
eventSource.addEventListener('thinking', (e) => {
    // 显示思考过程（可折叠）
    showThinking(e.data);
});

eventSource.addEventListener('message', (e) => {
    // 显示最终回复
    showMessage(e.data);
});
```

## 常见问题

### Q1: 为什么有时候还是会调用工具？
A: QuestionClassifier 采用保守策略，不确定的问题会归类为 COMPLEX。可以通过调整关键词列表来优化。

### Q2: 如何查看思考过程？
A: 前端监听 `thinking` 事件即可获取思考内容。

### Q3: 如何调整简单/复杂问题的判断？
A: 修改 `QuestionClassifier` 中的关键词列表。

### Q4: 思考链格式不对怎么办？
A: `ThinkingChainParser` 会容错处理，即使格式不对也能正常工作。
