# 工具调用优化说明 🔧

## 🎯 问题背景

### 用户反馈

用户要求查询地点信息，但 AI 没有调用工具，而是给出通用建议：

```
用户：帮我查一下广州的会约好地方

AI：广州啊...你倾向喜欢安静点的还是热闹点的？是想吃饭、散步，
    还是能一起干点啥的那种？顺便问一句，她是不是本地人？
    要是外地来的，可能更想看点有特色的；要是土生土长的那种，
    那些网红地她估计都去腻了。

问题：
- 没有调用高德地图工具查询
- 只是问了一堆问题
- 没有给出具体的地点信息
```

```
用户：我的另一半住在上海静安区，请帮我找到5公里内合适的约会地点

AI：等等，你刚说你在广州找约会地，现在又说她住上海静安区？
    你们是异地吗？还是你人现在在上海？要是异地的话，5公里范围
    其实真没啥意义——南京西路、静安寺一带，吃饭、散步、看展都能安排。
    但得先搞清楚：你们是打算在上海见她，还是你们都在上海？

问题：
- 没有调用地图工具查询
- 继续追问用户
- 没有提供具体地点
```

### 根本原因

1. **系统提示词中没有明确的工具使用指南**
   - 之前只有简单的"用户需要时默默处理"
   - 没有具体说明什么情况下应该使用什么工具

2. **AI 倾向于问问题而不是查询**
   - 系统提示词强调"信息不够就问"
   - 没有强调"主动使用工具查询"

3. **工具使用示例不足**
   - 没有展示如何正确使用地图工具的示例

## ✅ 优化方案

### 1. 添加"工具使用指南"

在系统提示词中添加了详细的工具使用指南：

```
工具使用指南
当用户有以下需求时，必须主动使用工具：

1. 地点查询（使用高德地图工具）
   - 用户问"哪里有xxx"、"推荐xxx地方"、"xxx附近有什么"
   - 用户要找餐厅、咖啡厅、电影院、公园等约会地点
   - 用户要查询路线、距离、周边设施
   - 示例：
     用户："帮我查一下广州的会约好地方"
     → 调用地图搜索工具，搜索广州的餐厅、咖啡厅等
     用户："上海静安区5公里内的约会地点"
     → 调用地图周边搜索工具，搜索静安区5公里内的约会地点

2. 信息查询（使用网络搜索工具）
   - 用户问最新的恋爱技巧、约会攻略
   - 用户要了解某个地方的特色、评价
   - 知识库中没有的信息

3. 文件操作（使用文件工具）
   - 用户要保存对话、报告
   - 用户要读取上传的文件

重要：不要只是建议用户去查，而是直接帮用户查询！
```

### 2. 添加工具使用示例

添加了正确使用工具的示例：

```
用户：帮我查一下广州的会约好地方

❌ 只给建议：
广州啊...你倾向喜欢安静点的还是热闹点的？是想吃饭、散步，
还是能一起干点啥的那种？

✅ 煎饼狗子：
[调用地图搜索工具查询广州的餐厅、咖啡厅、公园等]
我帮你查了一下，广州这几个地方不错：
1. 太古汇 - 高端商场，有很多餐厅和咖啡厅
2. 珠江新城花城广场 - 适合散步，晚上夜景很美
3. 沙面岛 - 欧式建筑，适合拍照
你们是第一次约会还是已经比较熟了？我可以根据你们的情况推荐更合适的。
```

### 3. 添加工具使用原则

```
工具使用原则
- 用户要查地点、路线 → 直接调用地图工具查询
- 用户要查信息、知识 → 直接调用搜索工具查询
- 用户要保存、读取文件 → 直接调用文件工具处理
- 不要说"你可以去查一下"，而是"我帮你查一下"
```

### 4. 添加禁止项

```
绝对禁止
- 不要只建议用户去查，要主动帮用户查
```

## 🔧 可用的工具

### 1. 高德地图工具（MCP）

恋爱大师已经集成了高德地图 MCP 工具，提供以下功能：

- 地点搜索（POI搜索）
- 周边搜索
- 路线规划
- 距离计算
- 地理编码
- 逆地理编码
- 等等...

### 2. DateLocationTool（本地工具）

提供恋爱场景专用的接口：

```java
@Tool(description = "搜索适合约会的地点")
public String searchDatePlaces(String location, String type)

@Tool(description = "规划约会路线")
public String planDateRoute(String from, String to, String mode)

@Tool(description = "查询某个地点周边的设施")
public String searchAroundDatePlace(String location, String radius, String type)
```

### 3. 其他工具

- `TavilyWebSearchTool` - 网络搜索
- `MailSendTool` - 邮件发送
- `FileOperationTool` - 文件操作
- 图片搜索 MCP 工具

## 💡 使用建议

### 1. 识别用户意图

当用户说以下内容时，应该调用地图工具：

```
✅ 明确的地点查询：
- "帮我查一下广州的会约好地方"
- "上海静安区5公里内的约会地点"
- "北京哪里有适合约会的咖啡厅"
- "推荐一些杭州的浪漫餐厅"

✅ 路线查询：
- "从天安门到故宫怎么走"
- "从我家到她家的路线"

✅ 周边查询：
- "这附近有什么好玩的"
- "静安寺周边有什么餐厅"
```

### 2. 工具调用流程

```
1. 识别用户意图
   ↓
2. 选择合适的工具
   ↓
3. 调用工具查询
   ↓
4. 整理查询结果
   ↓
5. 用自然语言回复用户
```

### 3. 回复格式

```
[调用工具查询]
↓
我帮你查了一下，[地区]这几个地方不错：
1. [地点1] - [描述]
2. [地点2] - [描述]
3. [地点3] - [描述]
↓
[可选：根据情况追问1个问题]
```
