# 问题分类优化说明 🔍

## 🎯 问题背景

### 之前的问题

用户发送"你好呀"时，日志显示：

```
问题分类结果: GENERAL, 问题: 你好呀
```

**为什么会这样？**

1. "你好呀"不包含任何恋爱关键词（如"恋爱"、"感情"、"女朋友"等）
2. "你好呀"也不包含通用问题关键词（如"天气"、"新闻"、"历史"等）
3. 所以被分类为 `UNKNOWN`（未知类型）
4. 在 `LoveApp.java` 中，`UNKNOWN` 类型和 `GENERAL` 类型一起处理
5. 因此日志显示为 `GENERAL`

### 处理逻辑

```java
// LoveApp.java 中的处理
switch (type) {
    case SENSITIVE:
        return "抱歉，我无法回答这个问题。";

    case LOVE_RELATED:
    case GENERAL:
    case UNKNOWN:  // ← 未知类型和通用类型一起处理
    default:
        return doChatWithRagOrFallback(message, chatId);
}
```

## ✅ 优化方案

### 方案：将问候语归类为恋爱相关

**理由**：

1. 恋爱大师是专门的恋爱咨询应用
2. 用户说"你好"通常是对话的开场白
3. 应该让用户感觉在和恋爱顾问对话，而不是通用 AI
4. 问候语应该触发恋爱相关的回复风格

### 实现方式

#### 1. 添加问候语关键词集合

```java
/**
 * 问候语集合（归类为恋爱相关，因为是恋爱大师的对话开场）
 */
private static final Set<String> GREETING_KEYWORDS = Set.of(
    "你好", "您好", "hi", "hello", "嗨", "哈喽", "早上好", "晚上好",
    "在吗", "在不在", "有人吗", "能帮我吗", "可以帮我吗"
);
```

#### 2. 修改分类逻辑

```java
public QuestionType classify(String question) {
    // ...

    // 1. 首先检测敏感词
    if (containsSensitiveKeyword(lowerQuestion)) {
        return QuestionType.SENSITIVE;
    }

    // 2. 检测问候语（归类为恋爱相关）← 新增
    if (containsGreetingKeyword(lowerQuestion)) {
        return QuestionType.LOVE_RELATED;
    }

    // 3. 检测恋爱关键词
    if (containsLoveKeyword(lowerQuestion)) {
        return QuestionType.LOVE_RELATED;
    }

    // 4. 检测通用问题关键词
    if (containsGeneralKeyword(lowerQuestion)) {
        return QuestionType.GENERAL;
    }

    // 5. 默认返回未知
    return QuestionType.UNKNOWN;
}
```

#### 3. 添加检测方法

```java
/**
 * 检测是否包含问候语
 */
private boolean containsGreetingKeyword(String question) {
    for (String keyword : GREETING_KEYWORDS) {
        if (question.contains(keyword)) {
            log.debug("检测到问候语: {}", keyword);
            return true;
        }
    }
    return false;
}
```

## 🎨 分类规则总结

### 优先级顺序

1. **敏感词检测**（最高优先级）
   - 包含敏感词 → `SENSITIVE`
   - 直接拒绝回答

2. **问候语检测**（新增）
   - 包含问候语 → `LOVE_RELATED`
   - 归类为恋爱相关对话

3. **恋爱关键词检测**
   - 包含恋爱关键词 → `LOVE_RELATED`
   - 恋爱相关问题

4. **通用问题关键词检测**
   - 包含通用关键词 → `GENERAL`
   - 通用问题（天气、新闻等）

5. **默认分类**
   - 都不匹配 → `UNKNOWN`
   - 未知类型

## 💡 进一步优化建议

### 1. 使用 AI 分类（更智能）

当前使用关键词匹配，可以改用 AI 模型分类：

```java
public QuestionType classifyWithAI(String question) {
    String prompt = """
        请判断以下问题的类型：
        1. LOVE_RELATED - 恋爱相关（包括问候语）
        2. GENERAL - 通用问题
        3. SENSITIVE - 敏感问题

        问题：%s

        只返回类型名称，不要解释。
        """.formatted(question);

    String result = chatClient.prompt()
        .user(prompt)
        .call()
        .content();

    return QuestionType.valueOf(result.trim());
}
```

**优点**：

- 更智能，能理解语义
- 不需要维护关键词列表
- 能处理复杂的问题

**缺点**：

- 需要调用 AI API，增加延迟
- 增加成本
- 可能不稳定

### 2. 混合方案（推荐）

```java
public QuestionType classify(String question) {
    // 1. 先用关键词快速判断
    QuestionType keywordResult = classifyByKeyword(question);

    // 2. 如果是 UNKNOWN，再用 AI 判断
    if (keywordResult == QuestionType.UNKNOWN) {
        return classifyWithAI(question);
    }

    return keywordResult;
}
```

**优点**：

- 大部分问题用关键词快速判断（低延迟）
- 复杂问题用 AI 判断（高准确率）
- 平衡了性能和准确性

### 4. 添加短问题特殊处理

```java
public QuestionType classify(String question) {
    // 短问题（<5个字）且不包含关键词，默认为恋爱相关
    if (question.length() < 5) {
        QuestionType result = classifyByKeyword(question);
        if (result == QuestionType.UNKNOWN) {
            log.debug("短问题默认归类为恋爱相关: {}", question);
            return QuestionType.LOVE_RELATED;
        }
    }

    return classifyByKeyword(question);
}
```
