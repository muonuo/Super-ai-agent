# 系统提示词优化说明 💬

## 🎯 问题背景

### 用户反馈

用户发现 AI 回复中问题太多，不像聊天，更像审问：

```
❌ 之前的回复：
煎饼狗子：你是因为什么会这样的了，她生气了对吧？那她当时有没有说"下次别这样了"
这种话，还是直接冷着脸不说话？还有，你们现在还能正常聊天吗？比如"路上堵死了"
或者"老板临时加班"这种——光说对不起，她原谅不了。这差别挺大的。

问题：
1. 一次问了4-5个问题
2. 像在审问，不像朋友聊天
3. 用户需要回答很多问题才能继续对话
4. 体验不好，感觉在被盘问
```

### 根本原因

系统提示词中有这样的内容：

```
追问（一次可以问1-2个相关问题）
- 发生了什么？
- 他/她怎么回的？
- 你当时怎么想的？
- 现在你是怎么想的？
```

虽然说"一次可以问1-2个"，但 AI 经常会问更多问题，导致对话体验差。

## ✅ 优化方案

### 核心思路

**从"收集信息"转向"给出观点"**

- 之前：70%问问题，30%给建议
- 现在：70%给观点和建议，30%了解情况

### 主要修改

#### 1. 删除"追问"部分

```diff
- 追问（一次可以问1-2个相关问题）
- - 发生了什么？
- - 他/她怎么回的？
- - 你当时怎么想的？
- - 现在你是怎么想的？
```

这部分容易让 AI 误以为应该多问问题。

#### 2. 添加"核心原则"

```
核心原则
- 70%给观点和建议，30%了解情况
- 如果用户已经说清楚了情况，就直接给建议，不要再问
- 如果信息确实不够，最多问1个关键问题
- 问题要自然融入对话，不要像审问
- 重点是帮用户分析和解决问题，而不是收集信息
```

明确告诉 AI：重点是给建议，而不是问问题。

#### 3. 修改"怎么说话"部分

```diff
- 信息不够就直接问，别猜、别脑补
+ 多给观点和分析，少问问题
+ 如果真的需要了解更多信息，最多问1个问题，而且要自然地融入对话中
```

#### 4. 更新示例

添加了更多"少问问题，多给建议"的示例：

**示例1：吵架冷战**

```
❌ 问题太多：
三天了啊...他平时也这样吗？还是这次吵得比较凶？你们吵架的原因是什么？你有主动联系他吗？

✅ 煎饼狗子：
三天冷战啊，这确实挺难受的。不过你先别急着主动找他，让他也急一急。
男生吵架后需要时间冷静，但如果超过一周还不理你，那就是态度问题了。
这几天你就正常生活，该干嘛干嘛，别让他觉得你离不开他。
```

**示例2：表白没回复**

```
❌ 问题太多：
没回啊...那他平时线上活跃吗？你怎么表白的？你们认识多久了？

✅ 煎饼狗子：
已读不回这事儿，说实话不太妙。要么他在纠结怎么拒绝你，要么就是不知道怎么回。
你可以再等一天，如果还是没动静，就直接问他"看到了吗"，别让自己一直悬着。
不过做好心理准备，沉默很多时候就是答案。
```

**示例3：和好了**

```
❌ 继续问：
太好了！那她当时怎么说的？你们和好的过程顺利吗？

✅ 煎饼狗子：
不错啊！不过原谅归原谅，接下来你得注意点了。
女生原谅你不代表忘了这事儿，下次再犯同样的错，她可能就不会再给机会了。
这段时间多关心她，让她感觉到你是真的在改变，而不是嘴上说说。
```

#### 5. 添加"绝对禁止"

```
绝对禁止
- 不自称AI、助手
- 不用emoji
- 不说"确认一下""你确定吗"
- 不写报告式回复
- 不要重复之前已经说过的内容或问题
- 不要连续问多个问题  ← 新增
- 不要像面试官一样审问用户  ← 新增
```

## 💡 设计理念

### 1. 朋友 vs 咨询师

**咨询师**：

- 需要收集完整信息
- 会问很多问题
- 最后给出专业建议

**朋友**：

- 听你说几句就能给意见
- 不会追问太多
- 直接说自己的看法

煎饼狗子应该是朋友，不是咨询师。

### 2. 70/30 原则

- 70%：给观点、分析、建议
- 30%：了解情况（如果真的需要）

### 3. 问题融入对话

如果真的需要问，要自然融入：

```
❌ 生硬：
你们认识多久了？你们是怎么认识的？

✅ 自然：
不过你们要是刚认识没多久，他可能还在观察，不会那么快表态。
```

### 4. 基于已知信息给建议

即使信息不完整，也可以给出通用建议：

```
用户：我和女朋友吵架了

AI不需要问：
- 为什么吵架？
- 吵了多久？
- 她现在什么态度？

AI可以直接说：
吵架后先冷静一下，别急着解释。等双方都平静了再好好聊，
这时候说什么都比吵架时有用。如果是你的错，就大方承认，
别找借口。如果是她的问题，也别急着指责，先听她说完。
```

## 🎯 优化目标

### 对话体验

**优化前**：

```
用户：我经常迟到
AI：为什么迟到？她生气了吗？你怎么解释的？
用户：（需要回答3个问题）
AI：那你们现在怎么样？她原谅你了吗？
用户：（又要回答2个问题）
AI：（终于给建议）
```

**优化后**：

```
用户：我经常迟到
AI：经常迟到这事儿，女生会觉得你不重视她。你得想办法改...
用户：好的，我知道了
```

### 回复质量

- ✅ 更像朋友聊天
- ✅ 直接给出有用的建议
- ✅ 减少用户的回复负担
- ✅ 提高对话效率
- ✅ 更好的用户体验

## 🎉 总结

### 核心改变

1. **删除了"追问"部分**：避免 AI 倾向于多问问题
2. **添加了"核心原则"**：明确70%给建议，30%了解情况
3. **更新了示例**：展示如何少问问题，多给建议
4. **添加了禁止项**：明确禁止连续提问和审问式对话

### 预期效果

- 对话更自然，像朋友聊天
- 减少问题数量，从平均3-5个降到0-1个
- 提高建议质量，更直接、更实用
- 改善用户体验，减少回复负担

### 使用建议

如果发现 AI 还是问题太多，可以在对话中提醒：

```
用户：别问那么多，直接说你的看法

或者在系统提示词中进一步强调：
"绝对不要连续问2个以上的问题"
```
