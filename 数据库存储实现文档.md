## 实现思路

### 1.1 需求分析
- **持久化需求**：将会话记录从内存/文件存储迁移到MySQL数据库
- **多轮对话支持**：支持同一会话ID的多轮对话记录
- **消息类型支持**：支持用户消息、助手消息、系统消息等多种类型
- **数据完整性**：确保消息内容、时间戳、元数据等信息的完整保存
- **性能优化**：使用批量操作提高数据库写入效率

### 1.2 技术选型
- **ORM框架**：选择MyBatis-Plus，相比JPA具有更好的性能和更简洁的API
- **数据库**：MySQL，成熟稳定的关系型数据库
- **Spring AI集成**：实现Spring AI的ChatMemory接口，无缝集成到现有架构

### 1.3 实现策略
1. **接口适配**：实现Spring AI的ChatMemory接口，保持与现有架构的兼容性
2. **分层设计**：采用经典的分层架构（Entity -> Mapper -> Repository -> Service）
3. **转换器模式**：使用专门的转换器处理Spring AI Message与数据库实体之间的转换
4. **批量操作**：利用MyBatis-Plus的批量操作提高性能
5. **自动填充**：使用MyBatis-Plus的自动填充功能处理时间字段

```java
public List<ChatMessage> listByConversationId(String conversationId) {
    LambdaQueryWrapper<ChatMessage> queryWrapper = new LambdaQueryWrapper<>();
    queryWrapper.eq(ChatMessage::getConversationId, conversationId)  // 等于指定会话ID
               .orderByAsc(ChatMessage::getCreateTime);              // 按创建时间升序排列
    return this.list(queryWrapper);
}
```

**2. 查询最新的N条消息**
```java
public List<ChatMessage> listLatestByConversationId(String conversationId, int limit) {
    LambdaQueryWrapper<ChatMessage> queryWrapper = new LambdaQueryWrapper<>();
    queryWrapper.eq(ChatMessage::getConversationId, conversationId)  // 指定会话ID
               .orderByDesc(ChatMessage::getCreateTime)              // 按创建时间降序排列
               .last("LIMIT " + limit);                            // 限制返回数量
    
    List<ChatMessage> chatMessages = this.list(queryWrapper);
    
    if (!chatMessages.isEmpty()) {
        Collections.reverse(chatMessages);  // 反转列表，按时间升序返回
    }
    
    return chatMessages;
}
```

**3. 删除指定会话的所有消息**
```java
public void deleteByConversationId(String conversationId) {
    LambdaQueryWrapper<ChatMessage> queryWrapper = new LambdaQueryWrapper<>();
    queryWrapper.eq(ChatMessage::getConversationId, conversationId);
    this.remove(queryWrapper);  // 使用逻辑删除
}
```

**设计优势**：
- 使用Lambda表达式，类型安全
- 封装常用查询逻辑，避免重复代码
- 支持逻辑删除，数据安全

### 3.4 MessageConverter 转换器

**文件位置**：`com.monuo.superaiagent.converter.MessageConverter`

**核心功能**：处理Spring AI Message与数据库实体之间的双向转换

#### 3.4.1 Spring AI Message转ChatMessage

```java
public static ChatMessage toChatMessage(Message message, String conversationId) {
    return ChatMessage.builder()
            .conversationId(conversationId)           // 设置会话ID
            .messageType(convertMessageType(message))     // 转换消息类型
            .content(getMessageText(message))           // 提取消息内容
            .metadata(message.getMetadata())           // 保留元数据
            .build();
}
```

**消息类型转换逻辑**：
```java
private static ChatMessage.MessageType convertMessageType(Message message) {
    if (message instanceof UserMessage) {
        return ChatMessage.MessageType.USER;
    } else if (message instanceof AssistantMessage) {
        return ChatMessage.MessageType.ASSISTANT;
    } else if (message instanceof SystemMessage) {
        return ChatMessage.MessageType.SYSTEM;
    } else {
        return ChatMessage.MessageType.USER;  // 默认为用户消息
    }
}
```

**消息内容提取逻辑**：
```java
private static String getMessageText(Message message) {
    if (message instanceof UserMessage userMessage) {
        return userMessage.getText();
    } else if (message instanceof AssistantMessage assistantMessage) {
        return assistantMessage.getText();
    } else if (message instanceof SystemMessage systemMessage) {
        return systemMessage.getText();
    } else {
        return "";
    }
}
```

#### 3.4.2 ChatMessage转Spring AI Message

```java
public static Message toMessage(ChatMessage chatMessage) {
    ChatMessage.MessageType messageType = chatMessage.getMessageType();
    String text = chatMessage.getContent();
    Map<String, Object> metadata = chatMessage.getMetadata();
    
    return switch (messageType) {
        case USER -> new UserMessage(text);
        case ASSISTANT -> new AssistantMessage(text);
        case SYSTEM -> new SystemMessage(text);
        case TOOL -> new UserMessage(text);  // TOOL类型转换为用户消息
    };
}
```

**设计优势**：
- 集中管理转换逻辑，便于维护
- 支持多种消息类型
- 保留元数据信息
- 使用Builder模式，代码简洁

### 3.5 DatabaseBasedChatMemory 核心实现

**文件位置**：`com.monuo.superaiagent.chatmemory.DatabaseBasedChatMemory`

**核心功能**：实现Spring AI的ChatMemory接口，提供会话记忆功能

#### 3.5.1 类结构

```java
@Component        // Spring注解，标记为Spring组件
@Slf4j           // Lombok注解，自动生成日志对象
@RequiredArgsConstructor  // Lombok注解，生成构造函数注入
public class DatabaseBasedChatMemory implements ChatMemory {
    
    private final ChatMessageRepository chatMessageRepository;  // 注入数据访问层
}
```

#### 3.5.2 添加消息实现

```java
@Override
public void add(String conversationId, List<Message> messages) {
    // 1. 使用Stream API转换消息
    List<ChatMessage> chatMessages = messages.stream()
            .map(message -> MessageConverter.toChatMessage(message, conversationId))
            .collect(Collectors.toList());
    
    // 2. 批量保存到数据库
    chatMessageRepository.saveBatch(chatMessages, chatMessages.size());
    
    // 3. 记录调试日志
    log.debug("Saved {} messages for conversation {}", messages.size(), conversationId);
}
```

**实现要点**：
- 使用Stream API进行函数式编程
- 批量操作提高性能
- 保留会话ID关联
- 记录操作日志便于调试

#### 3.5.3 获取消息实现

```java
@Override
public List<Message> get(String conversationId) {
    // 1. 从数据库查询消息
    List<ChatMessage> chatMessages = chatMessageRepository.listByConversationId(conversationId);
    
    // 2. 转换为Spring AI Message对象
    List<Message> messages = chatMessages.stream()
            .map(MessageConverter::toMessage)
            .collect(Collectors.toList());
    
    // 3. 记录调试日志
    log.debug("Retrieved {} messages for conversation {}", messages.size(), conversationId);
    
    // 4. 返回消息列表
    return messages;
}
```

**实现要点**：
- 按时间顺序返回消息
- 自动类型转换
- 支持完整的消息历史
- 记录查询日志

#### 3.5.4 清空消息实现

```java
@Override
public void clear(String conversationId) {
    // 1. 删除指定会话的所有消息
    chatMessageRepository.deleteByConversationId(conversationId);
    
    // 2. 记录调试日志
    log.debug("Cleared all messages for conversation {}", conversationId);
}
```

**实现要点**：
- 使用逻辑删除，数据安全
- 按会话ID批量删除


### 6.3 查询历史示例

```java
@Service
@RequiredArgsConstructor
public class ChatHistoryService {
    
    private final ChatMessageRepository chatMessageRepository;
    
    public List<ChatMessage> getConversationHistory(String conversationId, int limit) {
        return chatMessageRepository.listLatestByConversationId(conversationId, limit);
    }
    
    public List<ChatMessage> getAllConversations() {
        // 查询所有会话（按会话ID分组）
        LambdaQueryWrapper<ChatMessage> wrapper = new LambdaQueryWrapper<>();
        wrapper.select(ChatMessage::getConversationId)
               .groupBy(ChatMessage::getConversationId)
               .orderByDesc(ChatMessage::getCreateTime);
        return chatMessageRepository.list(wrapper);
    }
    
    public long getMessageCount(String conversationId) {
        LambdaQueryWrapper<ChatMessage> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(ChatMessage::getConversationId, conversationId);
        return chatMessageRepository.count(wrapper);
    }
}
```
